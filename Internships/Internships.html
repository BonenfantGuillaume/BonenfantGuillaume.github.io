Pharmaceutical data visualization

About the internship

This internship is a two-months internship made in a pair with another student in the Focussia company. The aim of this internship was at first to make a design for the company's project and then to start developing it, but knowing we wouldn't have the time to finish it because it's a big project that requires a lot of time.

The project

It was a pharmaceutical project for which we had to display data for machines that were preparing medicines, like the heat, number of turns per minute and more to facilitate looking for anomalies.

Our design

A big advantage of being a pair to work on this project was that because design is pretty subjetive, we could share and judge each other ideas pretty easily, and advance together. To think about and make the design of the project, we mainly used the balsamiq website. In our final design, the main design takes were to be able to add and delete as many charts as the user wants and to be able to personalize them, choosing which data to display, so for example the user would be able to have certain charts for certain data. Then the user would be able to select whether he wants to see real-time data or data on a specific date and time. He would also be able to make the chart full screen by clicking on it or going into a specific mode that would allow him to see the exact data for the lines where the cursor is pointing at.

Developing

For the development, we decided to use React which is a great choice to make single-page applications and to use Chart.js for our website's charts which seemed to be a great fit. We also used Jira to follow our progress as an agile team. We didn't have the time to develop a lot the web-app but still we have been able to do a few things like the top bar with real time being displayed, to build a first chart that was linked to a legend so the user could choose which data to display and which not to and also a search bar to look easily for the different data.

AI

About the internship

This internship was a one month internship in my university's lab. The aim was to learn the basics of AIs because I had no prior knowledge and wanted to learn a little about it. It actually looked more like a simple course one on one with a teacher rather than a real internship.

Project

At first I learnt the very basics of AIs, how they learn using backpropagation and how they make choices. Then I also learnt the basics of recurrent neural networks (RNN) through scientific papers. All this took about two weeks or so before starting the real project.
The aim of the real project was to build an AI that would be able to predict if a movie review is positive or negative according to the text of it. The teacher sent me a data sample of movie reviews and also an already existing Recurrent Neural Network and told me to try to understand it and adapt it. For these last two weeks I tried to understand the code behind what had been given to me and to see how I could use it in my project. I could eventually do something with it and have a decent result (around 80% accuracy if I remember well).

My opinion on this internship

My opinion on this internship is that my main goal which was learning the very basics of AIs was fulfilled. I might not have developped a lot on my own and not have built my own model but it is a pretty difficult task that was obviously not doable with no prior knowledge in this domain in one month.
The main negative point of it was that it was mainly a remote internship and I could barely see my internship master and it was hard to ask for help when it was needed. I also couldn't be in a working environment with other people around me which I would have greatly liked.
